# 实验二　按键实验(GPIO 输入)

本实验通过读取 GPIO 引脚的电平来检测按键状态,并在按键按下时翻转 LED,是学习 **GPIO 输入** 与 **按键消抖** 的入门例程。

- 实验平台:正点原子 ESP32-S3 开发板
- 按键连接引脚:`GPIO0`(BOOT 键,见 `components/BSP/KEY/key.h` 中的 `BOOT_GPIO_PIN`)
- 实验现象:每次按下 BOOT 键,LED 翻转一次(亮/灭切换)

---

## 一、GPIO 输入原理

### 1. 输入模式

与 LED 实验的输出模式相反,按键实验把引脚配置为**输入模式**:MCU 不主动驱动引脚,而是检测外部电路把引脚拉到的电平,再用 `gpio_get_level()` 读回 0 或 1。

### 2. 按键的电平变化

机械按键本质是一个开关,常见接法是**一端接地、一端接 GPIO**:

| 按键状态 | 引脚电平 | 读回值 |
| --- | --- | --- |
| 未按下 | 上拉到 3.3V(高) | 1 |
| 按下 | 接地(低) | 0 |

因此本实验属于**低电平有效**:检测到引脚为 `0` 即认为按键被按下(`BOOT == 0`)。

### 3. 上拉电阻的必要性

按键未按下时,引脚另一端悬空,电平不确定(浮空),读回值会随机跳变。为此必须给引脚一个确定的默认电平:

- 本实验使能**内部上拉**(`GPIO_PULLUP_ENABLE`),未按下时引脚被拉到高电平(1);
- 按下时按键将引脚直接接地,强制为低电平(0),覆盖上拉。

### 4. 软件消抖

机械触点在闭合/断开瞬间会有 5~10ms 的抖动,电平在 0/1 间快速跳变,若不处理会被误判为多次按下。常用的**软件消抖**方法:检测到按下后延时约 10ms,再次确认电平是否仍为低,确认后才认定为有效按下。

```c
if (key_boot && (BOOT == 0)) /* 检测到按下 */
{
    vTaskDelay(10);          /* 延时 10ms 消抖 */
    key_boot = 0;
    if (BOOT == 0)           /* 再次确认,排除抖动 */
        keyval = BOOT_PRES;
}
```

### 5. "只触发一次" vs "连续触发"

`key_scan()` 通过 `static` 标志位 `key_boot` 记录按键是否已松开,实现两种模式:

- **mode = 0(不支持连续按)**:按住不放只在第一次返回键值,必须松开再按才会再次触发——适合"按一次执行一次"的场景(本实验用此模式翻转 LED);
- **mode = 1(支持连续按)**:每次调用都强制复位标志,按住期间会持续返回键值——适合调节数值等需要长按连续生效的场景。

## 二、GPIO API 介绍

ESP-IDF 的 GPIO 驱动位于头文件 `driver/gpio.h`,按键实验主要用到以下接口。

### 1. `gpio_config()` —— 配置引脚为输入

```c
esp_err_t gpio_config(const gpio_config_t *pGPIOConfig);
```

| 成员 | 说明 | 本实验取值 |
| --- | --- | --- |
| `pin_bit_mask` | 要配置的引脚位掩码 | `1ull << BOOT_GPIO_PIN` |
| `mode` | 引脚模式 | `GPIO_MODE_INPUT`(纯输入) |
| `pull_up_en` | 内部上拉使能 | `GPIO_PULLUP_ENABLE` |
| `pull_down_en` | 内部下拉使能 | `GPIO_PULLDOWN_DISABLE` |
| `intr_type` | 中断触发类型 | `GPIO_INTR_DISABLE`(采用轮询扫描) |

> 与 LED 实验的区别:这里 `mode` 是 `GPIO_MODE_INPUT`(只读不写),且按"低电平有效"的接法使能了上拉。

### 2. `gpio_get_level()` —— 读取引脚电平

```c
int gpio_get_level(gpio_num_t gpio_num);
```

返回当前引脚电平(0 或 1)。本实验用宏封装,直接当作按键状态使用:

```c
#define BOOT gpio_get_level(BOOT_GPIO_PIN)   /* 0=按下, 1=松开 */
```

### 3. 轮询 vs 中断

本实验采用**轮询扫描**:在主循环里每 10ms 调用一次 `key_scan()` 检查电平。优点是简单直观;缺点是占用 CPU 且有扫描周期延迟。

进阶可改用**外部中断**(把 `intr_type` 设为 `GPIO_INTR_NEGEDGE` 下降沿触发,并用 `gpio_install_isr_service()` / `gpio_isr_handler_add()` 注册中断服务),按下瞬间立即响应,无需轮询。

### 4. 调用流程

```
app_main()
  ├── led_init()                 // 配置 GPIO1 为输出
  ├── key_init()                 // 配置 GPIO0 为输入 + 上拉
  └── while(1)
        ├── key = key_scan(0)    // 轮询扫描(不支持连续按)
        ├── if (key==BOOT_PRES) LED_TOGGLE();   // 按下则翻转 LED
        └── vTaskDelay(10ms)     // 每 10ms 扫描一次
```

## 三、硬件原理图

BOOT 键接在 `GPIO0` 上,典型连接如下(低电平有效):

```
        3.3V
         │
        ┌┴┐
        │ │  上拉电阻(可用内部上拉代替)
        └┬┘
         │
    GPIO0 ●────────► ESP32-S3(读取电平)
         │
         │  ┌──────┐
         └──┤ 按键 ├──── GND
            └──────┘
```

- **未按下**:按键断开,GPIO0 被上拉到 3.3V → 读回 **1**;
- **按下**:按键导通,GPIO0 直接接地 → 读回 **0**。

> ESP32-S3 的 GPIO0 同时是芯片的 **BOOT(启动模式选择)引脚**:复位时若该脚为低则进入下载模式,正常运行时可作普通输入使用。开发板已为其外接上拉,本实验又额外使能了内部上拉做双保险。

### 关键设计要点

1. **必须有确定的默认电平**:用内部或外部上拉/下拉避免引脚浮空误触发;
2. **务必消抖**:机械按键抖动会造成"一次按下多次响应",软件延时确认或硬件 RC 滤波均可;
3. **复用引脚注意**:GPIO0 是 BOOT 引脚,接外部电路时不要影响开发板正常上电启动。
